## 1. Objective

Build the dark-themed marketing section shown in the reference image: a serif headline
("The Three Pillars of a Review-Worthy App") above three stacked, rounded, bordered cards.
Each card has a gold/amber numbered title and a muted-gray body paragraph. The third card
carries a stronger gold glow/border to draw the eye.

Deliver it as a self-contained **mockup preview component** so it renders in the
`web-sandbox` Vite preview server with zero wiring.

---

## 2. Where It Lives (context discovered)

- `apps/web-sandbox` is a **component preview server** (Vite + React + Tailwind v4 + shadcn/ui).
- Mockup components live in `apps/web-sandbox/src/components/mockups/**/*.tsx`.
- `mockupPreviewPlugin.ts` auto-discovers every `.tsx` there (except `_`-prefixed files) and
  regenerates `src/.generated/mockup-components.ts`. **No manual registration needed.**
- A component is viewed at `/preview/<ComponentName>` (e.g. `/preview/ThreePillars`).
- The renderer (`App.tsx`) resolves the `default` export, then `Preview`, then the named
  export — so export the component as **default**.
- Theme tokens live in `src/index.css`. Dark mode is class-based (`.dark`), so the component
  must own its dark surface rather than relying on a global `.dark` toggle.
- Fonts are preloaded in `index.html` — **Playfair Display** (headline serif) and
  **Inter / DM Sans** (body) are already available. No font changes required.
- Utility `cn()` is at `@/lib/utils`; `@` aliases to `src`.

**Single file to create:**
`apps/web-sandbox/src/components/mockups/ThreePillars.tsx`

**Files auto-touched (do not edit by hand):**
`apps/web-sandbox/src/.generated/mockup-components.ts` (regenerated by the Vite plugin)

(amber accent, glow). Prefer `font-['Playfair_Display']` for the headline since the
sandbox loads fonts by family name; body inherits `font-sans`.

## 4. Content (verbatim from image)

**Headline:** The Three Pillars of a Review-Worthy App

1. **Instant Value Delivery** — Your app must deliver its core value within the first
   60 seconds of use. Users who experience a quick win are 4x more likely to leave a
   positive review. Do not gate your best features behind lengthy onboarding flows.

2. **Smooth User Experience** — Crashes, bugs, and confusing navigation are review killers.
   Before launch, test every flow in your app multiple times. A single crash can turn a
   potential 5-star reviewer into a 1-star complaint.

3. **Emotional Connection** — The apps that get the most enthusiastic reviews are the ones
   that make users feel something — whether that is accomplishment, relief, joy, or surprise.
   Build moments of delight into your app experience.

## 5. Implementation Steps

1. Create `apps/web-sandbox/src/components/mockups/ThreePillars.tsx`.
2. Define a `pillars` data array (`{ number, title, body }[]`) holding the three entries
   above — keeps JSX clean and makes the highlight rule (`isLast`) trivial.
3. Build the layout:
   - Root `<section>` with the dark background + warm radial glow, full-height centering.
   - Inner `<div className="mx-auto max-w-5xl ...">`.
   - `<h2>` headline with Playfair Display serif.
   - `.map()` over `pillars` into card `<article>`s; apply the amber glow variant when
     `index === pillars.length - 1`.
4. Export the component as **default** (`export default function ThreePillars() {...}`).
5. Match spacing, radius, and colors to the spec table; use `cn()` for the conditional
   highlight class.
